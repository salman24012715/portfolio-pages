Rpropensity_score_matching.R
# ── Propensity Score Matching (PSM) ───────────────────── # Gold standard for reducing selection bias in observational # surgical studies (e.g., laparoscopic vs open approach) library(MatchIt) library(cobalt) # 1. Perform 1:1 nearest-neighbor matching on logistic PS psm_match <- matchit( treat ~ age + sex + asa_score + bmi + comorbidity, data = df, method = "nearest", distance = "glm", # logistic regression for PS caliper = 0.2, # standardized caliper (Austin 2011) ratio = 1 ) # 2. Check covariate balance (SMD < 0.1 = good) love.plot(psm_match, vars = c("age", "sex", "asa_score", "bmi", "comorbidity"), thresholds = c(maha = .1), abs = TRUE ) # 3. Extract matched cohort & analyze matched_data <- match.data(psm_match) model_psm <- glm(complication_30d ~ treat, data = matched_data, weights = weights, family = binomial()) summary(model_psm)
Context: PSM is the workhorse of observational general surgery research—comparing outcomes between approaches (laparoscopic vs. open, robotic vs. conventional) when randomization is infeasible. Love plots for covariate balance and 0.2 standardized caliper are best practices per Austin (2011). Directly applicable to any NSQIP-based outcomes study.
Riptw_analysis.R
# ── Inverse Probability of Treatment Weighting (IPTW) ── # Retains full sample size (vs. matching which discards data) # Preferred when surgical cohort size is limited library(WeightIt) library(survey) # 1. Generate stabilized IP weights (ATE) wt_model <- weightit( treat ~ age + sex + asa_score + comorbidity + baseline_labs, data = df, method = "ps", estimand = "ATE" ) df$weights <- wt_model$weights # 2. Weighted outcome model via survey design design <- svydesign(ids = ~1, data = df, weights = ~weights) weighted_model <- svyglm(los_days ~ treat, design = design, family = gaussian()) summary(weighted_model) # 3. Verify balance after weighting love.plot(wt_model, thresholds = c(maha = .1))
Context: IPTW is preferred over PSM when sample size is limited (common in single-center surgical cohorts). It creates a pseudo-population where treatment assignment is independent of measured confounders, preserving statistical power by retaining every observation. Key for length-of-stay and complication rate comparisons.
SQL / DuckDBnsqip_outcomes.sql
-- Surgical outcomes from NSQIP-style registry -- 30-day complication rates by surgical approach and volume tier WITH surgeon_volume AS ( SELECT surgeon_id, COUNT(*) AS annual_cases, CASE WHEN COUNT(*) >= 50 THEN 'High-Volume' WHEN COUNT(*) >= 20 THEN 'Medium-Volume' ELSE 'Low-Volume' END AS volume_tier FROM procedures WHERE cpt_code IN ('44950', '47600', '49505', '44140') -- appendectomy, cholecystectomy, inguinal hernia, colectomy AND op_year = 2024 GROUP BY 1 ), outcomes AS ( SELECT p.*, sv.volume_tier, CASE WHEN p.days_to_complication <= 30 THEN 1 ELSE 0 END AS comp_30d FROM procedures p JOIN surgeon_volume sv USING (surgeon_id) ) SELECT volume_tier, approach, COUNT(*) AS n_cases, ROUND(AVG(comp_30d) * 100, 1) AS complication_pct, ROUND(AVG(los_days), 1) AS mean_los FROM outcomes GROUP BY 1, 2 ORDER BY volume_tier, complication_pct;
Context: Volume-outcome analysis is a major theme in general surgery health services research. This query stratifies 30-day complication rates by surgeon volume and surgical approach—the kind of analysis standard in JAMA Surgery and Annals of Surgery.
Rmeta_analysis_forest.R
# ── Random-Effects Meta-Analysis with Forest Plot ─────── # As applied to published surgical intervention RCTs library(metafor) library(meta) # Compute log odds ratios and variances from 2x2 data dat <- escalc( measure = "OR", ai = events_treat, n1i = n_treat, ci = events_ctrl, n2i = n_ctrl, data = study_data ) # Random-effects model with REML tau^2 estimator res <- rma(yi, vi, data = dat, method = "REML") summary(res) # I^2 heterogeneity and prediction interval cat("I^2:", res$I2, "\n") predict(res, transf = exp) # back-transform to OR scale # Publication-quality forest plot forest(res, slab = dat$study, xlab = "Odds Ratio (95% CI)", refline = 1, col = "#4fc3f7" ) # Egger's test for publication bias regtest(res, model = "lm")
Context: This is the exact workflow applied in two published meta-analyses (Asian J Psychiatry 2024; Eur J Gastroenterol Hepatol 2025). The REML estimator is the preferred tau² method per current meta-analytic guidelines. I² and prediction intervals quantify heterogeneity for surgical evidence-grading purposes.

Pythonsurgical_risk_prediction.py
# ── ML: Surgical Complication Risk Prediction ────────── # Random Forest with nested cross-validation & SHAP import pandas as pd import numpy as np from sklearn.ensemble import RandomForestClassifier from sklearn.model_selection import ( StratifiedKFold, cross_val_predict, GridSearchCV ) from sklearn.metrics import roc_auc_score import shap features = ['age', 'bmi', 'asa_score', 'operative_time_min', 'prior_abdominal_surg', 'diabetes', 'albumin', 'approach'] X, y = df[features], df['comp_30d'] # Nested CV: prevents optimistic AUC inflation outer_cv = StratifiedKFold(n_splits=5, shuffle=True, random_state=42) inner_cv = StratifiedKFold(n_splits=3, shuffle=True, random_state=42) rf = GridSearchCV( RandomForestClassifier(class_weight='balanced', random_state=42), {'n_estimators':[200,500], 'max_depth':[5,10]}, cv=inner_cv, scoring='roc_auc', n_jobs=-1 ) y_pred = cross_val_predict(rf, X, y, cv=outer_cv, method='predict_proba')[:, 1] print(f"AUC (nested CV): {roc_auc_score(y, y_pred):.3f}") # SHAP for interpretable feature importance rf.fit(X, y) explainer = shap.TreeExplainer(rf.best_estimator_) shap.summary_plot(explainer.shap_values(X)[1], X)
Context: ML-based surgical complication risk prediction is a rapidly growing field. Nested cross-validation prevents optimistic AUC inflation—a common methodological flaw in surgical ML papers. SHAP values provide clinically interpretable feature importance: BMI, ASA score, operative time, and prior abdominal surgery are standard predictors for general surgery 30-day outcomes.
